wcli0 — configuration source mockups

Static design mockups for letting the VS Code extension load, edit and save back configuration files (starting with .vscode/mcp.json), instead of only the current one-way new → export path.

The problem these explore

Today the panel always edits VS Code settings (wcli0.*, User or Workspace scope). Files are write-only outputs of the Export tab. There is no way to open an existing .vscode/mcp.json and edit it; round-tripping load → edit → save is impossible.

The unifying idea across all mockups is a new configuration source bar at the top of the panel. It always answers one question — “what am I editing right now, and where does Save go?” — and lets you switch the source: VS Code settings, the workspace .vscode/mcp.json, or any file you open explicitly.
